🛡️ Error Handling & Guardrails
  • Max Revision = 2 — ถ้า review fail 2 รอบ ให้ escalate to human
  • Dependency Lock — task ที่ depend on ผู้อื่นต้องรอ context ก่อนรัน
  • Fallback Plan — ถ้า Gemini ไม่ตอบ ให้ PM Claude ทำ architecture แทน
  • Token Budget — กำหนด max_tokens ต่อ task เพื่อควบคุมค่าใช้จ่าย

🔢 Priority Scoring (PM ใช้)

เมื่อมีหลาย task รอรัน PM จะเรียงลำดับตาม:

  • P0 (รันก่อน) — Architecture tasks ที่ tasks อื่น depend on
  • P1 — Core feature implementation
  • P2 — Supporting features, helpers
  • P3 (รันหลังสุด) — Tests, documentation, review
⚠️ Escalation Rules
  • Task ที่ Codex fail 2 ครั้ง → ส่งให้ Gemini แก้แทน
  • Architecture ที่ Gemini ตอบคลุมเครือ → Claude PM สรุป + ถาม user
  • Security issue severity HIGH → หยุด pipeline + แจ้ง user ทันที
  • Cost เกิน budget threshold → switch ไป mini/flash models
🛠️ Implementation Options
3 แนวทางที่เลือกได้ตามความถนัดและ infrastructure ที่มีอยู่
Option A — Python Orchestrator Script (แนะนำสำหรับเริ่มต้น)

script เดี่ยว run ได้เลย ไม่ต้องติดตั้งอะไรเพิ่ม เหมาะสำหรับทดสอบ concept

  • ✅ Setup ง่าย — แค่ pip install 3 packages
  • ✅ Debug ง่าย — เห็น log ทุก step
  • ✅ Flexible — แก้ routing logic ได้ทันที
  • ⚠️ ไม่มี UI — ต้องดูผ่าน terminal
Option B — n8n Workflow (เหมาะกับ Vinit ที่ใช้ n8n อยู่)

สร้าง visual workflow ใน n8n เชื่อม API ต่างๆ ผ่าน drag & drop

  • ✅ Visual flow — เห็นการทำงานแต่ละ node ชัดเจน
  • ✅ Error handling built-in — retry, webhook, notification
  • ✅ Schedule ได้ — ตั้งเวลารัน workflow อัตโนมัติ
  • ⚠️ ต้องมี n8n server — self-hosted หรือ cloud
Option C — FastAPI Service (Production-ready)

สร้าง microservice ที่ expose REST API รับ requirement และ return output

  • ✅ Scalable — deploy บน Render/Railway ได้เลย
  • ✅ Async — handle หลาย request พร้อมกัน
  • ✅ UI Integration — เชื่อมกับ React frontend ได้
  • ⚠️ Setup ซับซ้อนกว่า — ต้องมี backend knowledge

# main.py — AI Dev Team Orchestrator
# pip install anthropic openai google-generativeai python-dotenv

import asyncio, json, os
from dotenv import load_dotenv
from pm_claude import PMClaude
from dev_codex import CodexDev
from sr_gemini import GeminiSR

load_dotenv()

async def run_team(requirement: str):
    pm     = PMClaude()
    codex  = CodexDev()
    gemini = GeminiSR()
    results = {}

    # Phase 1: PM วิเคราะห์และวางแผน
    print("🤖 PM Claude: Analyzing requirement...")
    plan = await pm.analyze(requirement)
    print(f"📋 Project: {plan['project_name']} | {len(plan['tasks'])} tasks")

    # Phase 2: Execute tasks ตาม execution_order
    for task in plan["tasks"]:
        # Inject context from dependencies
        spec = task["spec"]
        for dep_id in task.get("depends_on", []):
            if dep_id in results:
                spec += f"\n\nContext:\n{results[dep_id]}"

        if task["assign_to"] == "codex":
            print(f"💻 Codex: {task['title']}")
            results[task["id"]] = await codex.implement(spec)
        else:
            print(f"🎓 Gemini: {task['title']}")
            results[task["id"]] = await gemini.work(spec, task["type"])

    # Phase 3: PM สร้าง final report
    report = await pm.final_report(plan, results)
    return report

if __name__ == "__main__":
    req = """สร้าง FastAPI backend สำหรับ SmartReceipt AI
    - OCR endpoint รับ PDF/Image
    - บันทึก PostgreSQL + JWT auth"""
    result = asyncio.run(run_team(req))
    print(result)

# pm_claude.py
import json
import anthropic

class PMClaude:
    def __init__(self):
        self.client = anthropic.Anthropic()
        self.model  = "claude-sonnet-4-6"

    async def analyze(self, requirement: str) -> dict:
        res = self.client.messages.create(
            model=self.model, max_tokens=2000,
            system="""You are a PM. Return ONLY valid JSON:
{
  "project_name": "...",
  "tasks": [{
    "id": "T001", "title": "...",
    "assign_to": "gemini|codex",
    "type": "architecture|code_gen|review",
    "spec": "detailed spec",
    "depends_on": []
  }]
}
Order: T001=gemini/architecture, T002-T003=codex/code_gen, T004=gemini/review""",
            messages=[{"role": "user", "content": requirement}]
        )
        text = res.content[0].text.replace("```json","").replace("```","").strip()
        return json.loads(text)

    async def final_report(self, plan: dict, results: dict) -> str:
        ctx = "\n".join(f"{k}: {v[:300]}" for k, v in results.items())
        res = self.client.messages.create(
            model=self.model, max_tokens=1000,
            system="You are a PM. Write a concise completion report in Thai/English.",
            messages=[{"role": "user",
                       "content": f"Project: {plan['project_name']}\nResults: {ctx}"}]
        )
        return res.content[0].text

# dev_codex.py
import openai

class CodexDev:
    def __init__(self):
        self.client = openai.AsyncOpenAI()

    async def implement(self, task_spec: str,
                          model: str = "gpt-4o") -> str:
        res = await self.client.chat.completions.create(
            model=model,
            messages=[
                {"role": "system",
                 "content": "You are a Senior Developer. Write "
                           "production-ready code only. Include docstrings "
                           "and type hints. Max 60 lines per file."},
                {"role": "user", "content": task_spec}
            ],
            max_tokens=2000, temperature=0.2
        )
        return res.choices[0].message.content

    async def implement_simple(self, task_spec: str) -> str:
        # ใช้ mini model สำหรับ boilerplate ประหยัด cost
        return await self.implement(task_spec, model="gpt-4o-mini")

# sr_gemini.py
import google.generativeai as genai
import os

class GeminiSR:
    PROMPTS = {
        "architecture": "You are a Senior Architect. Design: "
            "Tech Stack, DB Schema, API structure, 2 recommendations.",
        "review": "You are a Code Reviewer. Provide: "
            "3 strengths, 3 issues, 2 security notes, rating/10.",
        "security": "You are a Security Engineer. Audit for "
            "OWASP Top 10, injection, broken auth, sensitive data.",
        "performance": "You are a Performance Engineer. "
            "Identify bottlenecks and suggest optimizations."
    }

    def __init__(self):
        genai.configure(api_key=os.environ["GEMINI_API_KEY"])
        self.pro   = genai.GenerativeModel("gemini-2.5-pro")
        self.flash = genai.GenerativeModel("gemini-2.5-flash")

    async def work(self, spec: str, task_type: str) -> str:
        system = self.PROMPTS.get(task_type, self.PROMPTS["review"])
        # Flash สำหรับ review, Pro สำหรับ architecture
        model = self.flash if task_type == "review" else self.pro
        res = model.generate_content(f"{system}\n\nTask: {spec}")
        return res.text

📋 Statement of Work (SOW)
ขอบเขตงานและเงื่อนไขสำหรับแต่ละ AI agent — ใช้เป็น system prompt หรือ contract specification
🤖
SOW — Claude Code (PM)
project_manager · orchestrator
PM

📌 Scope of Work

  • รับ requirement จาก user และวิเคราะห์ให้เป็น structured task plan (JSON)
  • แตก requirement เป็น tasks ไม่เกิน 8 tasks ต่อ session พร้อม priority และ dependency
  • กำหนด assign_to (codex/gemini) สำหรับแต่ละ task ตาม routing rules
  • เขียน spec ที่ชัดเจนสำหรับแต่ละ task พร้อม context ที่จำเป็น
  • รอรับ output จากทุก agent และรวมเป็น final report
  • ตรวจสอบว่า output ครบถ้วนตาม requirement เริ่มต้น
  • แจ้ง user หากพบ blocker หรือ requirement ไม่ชัดเจน

🚫 Out of Scope

  • เขียนโค้ด implementation (ยกเว้น final integration)
  • ออกแบบ architecture ด้านเทคนิค (มอบหมายให้ Gemini SR)
  • Deploy หรือ run โค้ดบน server จริง

✅ Acceptance Criteria

  • Task plan ที่ valid JSON parse ได้ทุกครั้ง
  • แต่ละ task มี id, title, assign_to, type, spec ครบ
  • Final report ระบุ deliverables, next steps, risks ได้ชัดเจน

SOW — OpenAI Codex (Developer)
developer · code_gen specialist
DEV

📌 Scope of Work

  • รับ task spec จาก PM และเขียนโค้ดที่ production-ready ตาม spec ทุกรายการ
  • เขียน FastAPI endpoints พร้อม type hints, docstrings, error handling
  • สร้าง SQLAlchemy models, Pydantic schemas, Alembic migration
  • เขียน React/Next.js components พร้อม TypeScript และ loading/error states
  • ติดตั้ง JWT auth middleware, OAuth2 flow, refresh token logic
  • สร้าง unit tests (pytest/Jest) ที่ cover happy path และ edge cases
  • เขียน Dockerfile, docker-compose.yml, GitHub Actions CI/CD
  • ส่ง output เป็น code blocks พร้อม filename และ brief explanation

📏 Code Standards

  • PEP8 สำหรับ Python / ESLint standard สำหรับ JavaScript
  • ไม่มี hardcoded credentials — ใช้ environment variables เสมอ
  • แต่ละ file ไม่เกิน 150 lines — refactor เป็น modules ถ้าเกิน
  • ทุก public function ต้องมี docstring อธิบาย params และ return

✅ Acceptance Criteria

  • โค้ด syntax ถูกต้อง — ไม่มี SyntaxError เมื่อ parse
  • ครบตาม spec ที่ PM กำหนด — ไม่ขาด feature ที่ระบุ
  • Import statements ครบ — ไม่มี undefined import
⚠️ Revision Policy: ถ้า output ไม่ผ่าน acceptance criteria ให้ retry สูงสุด 2 ครั้ง หลังจากนั้น escalate to Gemini SR

SOW — Gemini Pro (Senior Architect)
senior_architect · reviewer · security
SR

📌 Scope of Work — Architecture

  • ออกแบบ system architecture ที่ scalable, maintainable, secure
  • เลือก tech stack พร้อมเหตุผลที่ชัดเจน (performance, cost, team skill)
  • ออกแบบ database schema — tables, indexes, foreign keys, constraints
  • กำหนด API contract — endpoints, request/response format, error codes
  • วาง folder structure ที่ follow best practices ของ framework ที่เลือก
  • ระบุ potential bottlenecks และ วิธี scale ในอนาคต

🔍 Scope of Work — Code Review

  • ตรวจ correctness — logic errors, off-by-one, null pointer, edge cases
  • ตรวจ security — SQL injection, XSS, CSRF, broken auth, sensitive data exposure
  • ตรวจ performance — N+1 queries, missing indexes, inefficient loops
  • ตรวจ maintainability — code duplication, naming, complexity
  • ให้ rating 1-10 พร้อม justification
  • ระบุ critical issues (ต้องแก้) vs suggestions (ควรแก้)

✅ Acceptance Criteria

  • Architecture doc มี Tech Stack, Schema, API Contract, Recommendations ครบ
  • Review report ระบุ severity level (Critical/High/Medium/Low) ทุก issue
  • ข้อเสนอแนะเป็น actionable — ระบุว่าแก้อย่างไร ไม่ใช่แค่บอกว่ามีปัญหา
